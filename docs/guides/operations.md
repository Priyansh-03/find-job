# Operations

## Daily run

- Use `./run.sh` for local execution.
- Use dashboard to trigger user-specific scrapes.
- Check `output/jobs_<Name>.csv` for persisted results.

## Troubleshooting

- **No rows in UI after run:** verify scrape log in dashboard and inspect user CSV in `output/`.
- **Render deploy fails on root path:** keep Root Directory blank.
- **Gunicorn not found:** ensure build command installs `requirements.txt`.
- **Source returns irrelevant rows:** check location hint and source keyword adaptation.

## Maintenance

- Refresh board slugs periodically:
  - `.venv/bin/python discover_boards.py`
- Probe board health:
  - `.venv/bin/python tests/probe_board_slugs.py`
- Keep source options aligned with:
  - `docs/FETCHER_OPTIONS.md`

## Logging and artifacts

- Output data: `output/`
- Runtime logs: `logs/` (if generated by current run path)
- Board inputs: `job_boards/`
